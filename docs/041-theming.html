13  Theming html

  • Parce que c’est un support de formation sur le bonheur, on regardera principalement les éléments les plus visuels comme les couleurs.
  • On restera ici sur les thèmes bootstrap libres, mais je conseille de jeter un coup d’oeil à certaines extensions comme celle de Julien Barnier qui permet d’utiliser un thème très sympa, avec un mode light et dark [Lien vers l’extension].

13.1 Les 26 thèmes bootstrap

  • Les 26 thèmes bootstrap libres sous la norme actuelle html (5) sont directement utilisables dans Quarto.
  • Dans le yaml du document ou du fichier _quarto.yml on change le thème en option du format html: theme: nom_theme.
---
format: 
  html:
    theme: nom_theme
---

Si on souhaite utiliser le thème zephyr:

---
format: 
  html:
    theme: zephyr
---

Normalement tout le monde peut trouver son bonheur avec ces thèmes, du plus classique (default) au plus psychédelique (quartz). On peut néanmoins vouloir à moindre frais modifier certains éléments visuels.

13.1.1 Thèmes light/dark

  • Dans les 26 thèmes, un même thème a une variante light et une variante dark:
    • Thème light: flatly.
    • Thème dark: darly.
  • Avec Quarto, on peut utiliser switcher de l’un à l’autre.
  • L’option theme aura donc deux thèmes en sous option:
    • light: nom_theme.
    • dark: nom_theme.
  • Au delà de ces deux thèmes clé en main, on peut créer un thème de type dark à partir d’un thème light existant, et inversement.
  • On devra utiliser un fichiers sccs et renseigner manuellement les affectations des couleurs.
  • Avec des variables SASS propres à Quarto, on peut faire ces modifications assez facilement. Attention toutefois à bien maîtriser l’architecture des couleurs entrant dans un thème html de type bootstrap.

Si on souhaite générer une version dark du thème zephyr:

  • On utilisera un fichier .scss qui affectera les couleurs pour obtenir un thème dark, par exemple zephyr_dark.scss.
  • On l’ajoutera à l’option dark de la façon suivante.
---
format: 
  html:
    theme:
      light: zephyr
      dark: [zephyr, zephyr_dark.scss]
---
Deux thèmes light ou dark

A l’utilité contestable, mais on pourrait hésiter entre deux thèmes lors de la création de la documantation. On pourrait utiliser et switcher entre deux thèmes de type light ou dark.

Par exemple, on peut utiliser simultanément les thèmes light zephyr et yeti:

---
format: 
  html:
    theme:
      light: zephyr
      dark:  yeti
---

13.1.2 Changer directement des éléments du thème dans le yaml

Sans passer par des manipulations de styles css dans le document, Quarto met à disposition des options qui s’appliquent directement au yaml pour modifier des éléments du thème, comme la couleur des liens Quelques options

Pour une simple question d’homogénéité avec les couleurs prédéfinies du thème il est préférable d’utiliser les couleurs du thème qui a été sélectionné.

Ces couleurs sont disponibles dans un fichier .scss du thème. Il est très facile de les récupérer, via le site bootswatch ou via des mises à disposition sur un dépôt git:

  • Par Quarto.
  • Par ce support.

Avec les couleurs du thème zephyr:

Pour modifier la couleur du lien (avec une couleur du thème), on utilise directement l’option linkcolor:

---
format: 
  html:
    theme: zephyr
    linkcolor: "#d63384"
---

Après un peu d’entraînement il est néanmoins préférable et bien plus facile de passer par un fichier scss et les variables SASS de Quarto pour réaliser ce type de modification.

Organisation d’un fichier .scss

  • Avec quarto, une série de variables SASS prédéfinies permettent de modifier automatiquement le style du document.

<variables sass>
  • Une Zône de création des classes css, optionnelle.
/*-- scss:rules --*/

<styles css>

/*-- scss:defaults --*/

$variable_sass2: attribut

$variable_sass_quarto: attribut   //affectera directement le style du document

/*-- scss:rules --*/

.classe_css{attribut1:$var_sass1; attribut2: $var_sass2;....}

13.2.3 Modification du thème avec les variables Sass de Quarto

Note

On peut appliquer un fichier .scss au format reveals.js. Comme le nombre de variables Sass Quarto pour modifier un thème revealsjs est plus limité que pour les documents [lien], site… on peut se faire la main sur les présentations dans un premier temps.

  • Les variables Sass de Quarto permettent pratiquement à elles seules de faire un thème complet pour un document ou un site, ou de modifier drastiquement un thème bootstrap existant.

  • l’ensemble des ces variables, dont le nombre augmente à chaque nouvelle version, sont directement récupérables sur le [site officiel]

  • Aucune difficulté ici, pour modifier un élement du thème il suffit d’affecter un attribut à ces variables, à partir d’une variable Sass précédemment définie ou d’un attribut quelconque: couleur, taille, épaisseur….

Exemple: la couleur de fond d’une page

$body-bg: couleur;

$body-bg: red;
// Une variable sass d'un thème bootstrap
$body-bg: $warning;

13.2.4 Ajout de styles css dans un fichier scss

  • Dans la section /*-- scss:rules --*/ du fichier, on peut créer des styles css en utiisant les variables Sass générées précédemment. Par exemple pour changer seulement la couleur du texte avec 8 couleurs Sass du thème bootstrap:

/*-- scss:defaults --*/

// liste des variables Sass: $couleurs1.... => $primary, $secondary,....

/*-- scss:rules --*/
.col_primary{color: $primary}
.col_secondary{color: $secondary}
.col_success{color: $success}
.col_info{color: $info}
.col_warning{color: $warning}
.col_danger{color: $danger}
.col_light{color:$light}
.col_dark{color: $dark}

On peut alors ajouter ces styles à des éléments du documents: mot, phrases, paragraphes….

On change la couleur du [mot success]{.col_success} avec la classe `.success`.

  • Un paragraphe avec une section div ::: {.nom_classe}:
::: {.warning}
On change la couleur du texte du paragraphe avec la classe `.warning`.
:::

Donne le résultat suivant:

```{.sass}
.col_primary{color: $primary}
.col_secondary{color: $secondary}
.col_success{color: $success}
.col_info{color: $info}
.col_warning{color: $warning}
.col_danger{color: $danger}
.col_light{color:$light}
.col_dark{color: $dark}
```

On peut alors ajouter ces styles à des éléments du documents: mot, phrases, paragraphes....

* Un mot ou une phrase avec un shortcode **[texte]{.nom_classe}**:

```{.md}
On change la couleur du [mot success]{.col_success} avec la classe `.success`.
```

Donne le résultat suivant:  

::: {.border}
On change la couleur du [mot success]{.col_success} avec la classe `.success`.
:::

<br>

* Un paragraphe avec une section div **::: {.nom_classe}**: 

```{.md}
::: {.warning}
On change la couleur du texte du paragraphe avec la classe `.warning`.
:::
```

Donne le résultat suivant: 

::: {.col_warning}
On change la couleur du texte du paragraphe avec la classe `.warning`.
:::
